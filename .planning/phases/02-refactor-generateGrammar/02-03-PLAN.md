实现：

```python
def learn(self, token_sequence, initial_tokens, state_features, state_dependencies, participant_file_names, participant_ids) -> GrammarLearningResult
def detect_skip_gram(self, result: GrammarLearningResult, n_positions: np.ndarray) -> SkipGramResult
```

`learn()` 必须复刻旧默认 `Chunking()` 语义：
1. `active_tokens` 从旧 `S` 顺序开始。
2. `original_sequence` 保存删除 `N` 后的原始 token 序列。
3. 每轮用当前 `parsed_sequence` 组织离散数据。
4. child 排除 `("V", "1", "2", "N", "S", "e")`。
5. parent 排除 `("V", "N")`。
6. 若 parent 与 child 的基础 token 有交集，跳过。
7. 用 `bd_score()` 计算无 parent score 和有 parent/condition score。
8. 过滤 `U[1,1] / len(parsed_sequence) < probability[parent] * probability[child]`。
9. 过滤 `U[1,1] / len(parsed_sequence) < 0.05`。
10. 选择 ratio 大于 1 且接近最大 ratio 的候选。
11. 用 `combine_tokens(parent, child)` 生成新 token，例如 `"G-L"`。
12. 用最长匹配重新解析 `original_sequence`。
13. 最近 5 次 KL 均值不超过 `0.05` 时停止。

`detect_skip_gram()` 必须复刻旧 `skip_gram()`：
1. 根据 `parsed_sequence` 中每个 token 的 `token_length()` 重建删除 `N` 前后位置。
2. 插回 `"N"`。
3. 对每个 `"N"`，检查后续第 2 到第 5 个 token 是否等于参数 `skip_gram_target`，默认 `"E-A"`。
4. 用 `bd_score()` 比较无 parent 和以 `N` 为 parent 的结果。
5. 若 `score2 / score1 > 1` 且 `U[1,1] / len(new_sequence) > 0.025`，返回 `SkipGramResult(True, U[1,1])`，否则返回 `SkipGramResult(False, 0)`。
</action>
