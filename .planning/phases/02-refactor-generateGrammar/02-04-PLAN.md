# Plan 02-04: 实现 legacy/structured 输出、pipeline 和运行脚本

<objective>
把 grammar 核心接入文件级处理流程，生成顶层包含 `legacy` 和 `structured` 两个字典的新 pickle，并提供 `script/run_generate_grammar.py` 运行入口。`legacy` 负责旧字段精确兼容，`structured` 负责清晰的新结构。
</objective>

<context>
本计划是核心算法到用户可运行入口的连接层。旧占位符只能出现在 `legacy.py`，不能回流到 `GrammarLearner`。
</context>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| 新输出不能与旧 pickle 逐 key/value 比较 | HIGH | `legacy` 必须包含旧 13 个字段：`sets`、`pro`、`gram`、`sequence`、`time_pro`、`frequency`、`seq`、`state`、`S`、`fileNames`、`components`、`skipGram`、`skipGramNum`。 |
| legacy 兼容层污染核心算法 | MEDIUM | 旧占位符生成函数只放在 `legacy.py`，`grammar.py` 不出现 `place_set`。 |
| 运行脚本误写原项目输出目录 | HIGH | 默认输出必须写到 `data/generate_grammar/refactored-output/grammar2/`。 |
</threat_model>

<action>
创建 `legacy.py`。

实现：

```python
LEGACY_FIELD_ORDER = ("sets", "pro", "gram", "sequence", "time_pro", "frequency", "seq", "state", "S", "fileNames", "components", "skipGram", "skipGramNum")

def build_legacy_output(result: GrammarLearningResult, skip_gram: SkipGramResult) -> dict[str, Any]
```

`build_legacy_output()` 必须：
1. 返回普通 `dict`，字段插入顺序等于 `LEGACY_FIELD_ORDER`。
2. `sets` 使用旧格式，复合 token 去掉 `-`，例如 `"G-L-E-A"` -> `"GLEA"`。
3. `sequence` 使用删除 `N` 后的原始序列字符串，例如 `["G", "L"]` -> `"GL"`。
4. `fileNames` 使用 `result.participant_file_names`，保留 `.pkl` 后缀。
5. `skipGram` 使用 `skip_gram.found`。
6. `skipGramNum` 使用 `skip_gram.count`。
7. 为旧字段 `S` 和 `seq` 临时生成旧版占位符映射。

旧版占位符顺序必须复刻旧代码：

```python
place_set = [chr(i) for i in range(32, 126)]
for token in ("e", "G", "L", "E", "A", "1", "2", "3", "4", "S", "V", "N"):
    place_set.remove(token)
```

兼容层可以生成占位符，但 `legacy.py` 之外不能使用这些占位符作为核心算法状态。
</action>

<action>
创建 `pipeline.py`。

定义：

```python
@dataclass
class PreparedStrategyStateData:
    input_file_name: str
    token_sequence: list[str]
    n_positions: np.ndarray
    initial_tokens: list[str]
    state_features: pd.DataFrame
    participant_file_names: list[str]
    participant_ids: list[str]
    state_dependencies: StateDependencyGraph

def prepare_strategy_state_data(data: StrategyStateData, state_dependencies: StateDependencyGraph, removed_token: str = "N") -> PreparedStrategyStateData
def process_strategy_state_file(input_file_name: str, config: GenerateGrammarConfig) -> dict[str, Any]
def run_generate_grammar(config: GenerateGrammarConfig) -> list[Path]
```

`prepare_strategy_state_data()` 必须删除 `removed_token`，记录 `n_positions`，并同步删除 `state_features` 中对应行。

`process_strategy_state_file()` 必须返回：

```python
{
    "legacy": legacy_dict,
    "structured": structured_dict,
}
```

`run_generate_grammar()` 必须：
1. 调用 `config.validate()`。
2. 排序枚举输入 `.pkl` 文件。
3. 对每个文件调用 `process_strategy_state_file()`。
4. 写入 `config.output_dir / input_file_name`。
5. 返回输出路径列表。
</action>
