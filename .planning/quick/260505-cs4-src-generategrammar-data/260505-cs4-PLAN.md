# Quick 任务计划：移除 src 中旧项目数据目录依赖

## 目标

新版本 `generate_grammar` 不能再依赖 Pacman 旧项目的代码目录或数据目录。当前重构实际需要的输入数据和旧基准数据必须迁移到 LoPS 仓库的 `data/generate_grammar` 下；`src/LoPS` 中不再保存任何旧项目绝对路径，也不再为输入路径设置默认值。

## 数据目录

- 输入策略序列：`data/generate_grammar/input/strategy_sequence`
- 输入状态图：`data/generate_grammar/input/state_graph`
- 旧输出基准：`data/generate_grammar/baseline/grammar`
- 新输出：`data/generate_grammar/refactored-output/grammar`
- smoke 输出：`data/generate_grammar/smoke-output`

## 实施步骤

1. 复制本轮实际使用的 34 个 `StrategySequence`、34 个 `StateGraph` 和 34 个 `grammar2` 基准文件到 `data/generate_grammar`。
2. 修改 `GenerateGrammarConfig`：输入路径和输出路径由调用方显式传入，`src` 不再定义外部路径默认值。
3. 修改运行脚本和验证脚本：脚本层可定义 LoPS/data 下的路径常量，但构造配置时必须显式传入。
4. 修改测试：测试只使用 `data/generate_grammar` 下的数据，不再从 `src` 常量或旧 Pacman 目录读取数据。
5. 更新数据说明和 GSD 状态。
6. 运行单元测试、脚本级 smoke test 和一致性验证。

## 非目标

- 不修改核心 grammar 学习算法。
- 不修改旧 Pacman 项目。
- 不把 `pkl` 数据纳入 git 跟踪；数据仍由 `.gitignore` 作为运行/验证数据忽略。
