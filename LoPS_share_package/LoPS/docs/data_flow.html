Pacman 数据处理流程说明

本文档说明当前 LoPS 仓库中的 Pacman 数据处理流程。主分析流程按照 script/ 根目录中编号脚本的执行顺序组织；视频流程保留在 script/pacman_video/ 下，作为独立渲染链路。

当前默认数据根目录为 data/。主分析流程从 data/00_raw_mat_data 开始，逐阶段写入带编号的数据目录。

主分析数据文件统一使用 {subject/session}.pkl 命名，例如 031222-401-03-Dec-2022-1.pkl。阶段信息只体现在目录名中，不再写入文件名。

主分析流程

主分析流程只考虑两只鬼的数据。每个阶段尽量读取上一阶段输出，生成自己负责的新增字段或结构； 已经标准化后的字段会在后续阶段一致保留，避免重复解释和重复转换。

前三个阶段的可复用代码位于 src/LoPS/pacman_preprocess/。 该模块名强调其功能是 Pacman 原始行为数据预处理，而不是泛指所有 Pacman 数据。

01. raw mat 到 raw subject data

脚本 script/01_mat_to_raw_subject_data.py

输入数据data/00_raw_mat_data 下的原始 Pacman .mat session 目录。
输出数据data/01_raw_subject_data 下每个 session 一个 {session}.pkl。

目的

将原始 MATLAB 数据整理为 Python 后续流程可直接读取的单被试逐 trial 表格。 该阶段保留原始记录中的位置、方向、模式、水奖励等基础字段，为 frame data 构建提供完整来源。

处理方式

1. 发现 session 并并行调度

脚本扫描 data/00_raw_mat_data 下的 session 目录。若命令行传入 session 白名单， 只处理指定 session；否则处理全部目录。每个 session 是一个独立任务，可以通过 --workers 并行。

2. 读取 MATLAB 原始结构

每个 session 内读取原始 .mat 数据，把 MATLAB table 中的逐帧记录转为 pandas 可处理的数据表。该阶段不做行为分析，只负责把原始采集字段搬到当前仓库的数据格式中。

3. 保留 raw 级基础字段

输出中保留后续 frame 构建需要的原始列，例如 DayTrial、Step、 Map、pacMan_1/2、ghost 坐标和状态、方向、帧编号、水奖励字段等。

4. 写出 session 级 pickle

每个 session 写成一个 {session}.pkl。文件仍然接近原始采集形态， 下游会在 frame 阶段再做去重、排序、过滤和坐标列表解析。

02. raw subject data 到 frame data

脚本 script/02_raw_subject_data_to_frame_data.py

输入数据data/01_raw_subject_data。
输出数据data/02_frame_data，可选 CSV 输出到 data/02_frame_data_csv。

目的

将逐 trial 原始记录转换为逐 frame 的基础数据表。该阶段生成统一的 frame_id，并且只保留两只鬼 trial；四只鬼 trial 会被过滤掉。

处理方式

1. 每帧去重

输入 raw subject data 可能存在同一 DayTrial-Step 多行记录。脚本按 DayTrial 和 Step 分组，并取每组首行，保证输出中每一行对应唯一 frame。

2. 过滤四鬼 trial

当前主分析流程只考虑两只鬼。脚本在 frame data 生成阶段按整局判断并过滤四鬼 trial， 避免后续地图常量和 utility 计算遇到当前流程不支持的位置或策略维度。

3. 数字顺序排序

输出前先按 DayTrial 的数字段和 Step 排序，而不是直接用字符串排序。 这样 2-1 会排在 10-1 前面，保证 frame_id 稳定。

4. 构造 frame 字段

Pacman 和 ghost 的格点位置会被整理成 tuple[int, int]；Map 字符串会被解析出 beans 和 energizers 坐标列表；渲染仍需使用的像素坐标、方向、帧编号和水奖励字段继续保留。

5. 生成 frame_id 并写出

排序完成后生成整数 frame_id，替代旧流程里的 Unnamed: 0。 输出文件名继续使用 {session}.pkl；只有传入 --write-csv 时才额外写 CSV。

03. frame data 预处理

脚本 script/03_frame_data_preprocess.py

输入数据data/02_frame_data。
输出数据data/03_preprocessed_frame_data。

目的

生成后续分析使用的标准 frame 表。该阶段是字段清理和标准化入口， 将 raw frame 中的分析必要字段整理成更小、更稳定的数据形态。

处理方式

1. 读取 raw frame 并校验必要字段

输入必须包含 DayTrial、Step、pacmanPos、 ghost1Pos、ghost2Pos、ifscared1/2、beans 和 energizers。缺少任一字段会直接报错。

2. 标准化 id 和 trial 字段

frame_id 被强制转为 int64；DayTrial 转为字符串； game_id 从 DayTrial 中去掉第二段 trial round 得到。例如 1-2-031222-401-03-Dec-2022 会生成 1-031222-401-03-Dec-2022。

3. 标准化坐标和列表

pacmanPos、ghost1Pos、ghost2Pos 被转为 tuple[int, int]。beans 和 energizers 被转为 list[tuple[int, int]]，缺失或空值按空列表处理。

4. 标准化 scared 状态

ifscared1 和 ifscared2 转为 int8； 缺失值用 -1 表示不存在或不可用状态。

5. 收敛为分析字段集合

输出只保留 frame_id、DayTrial、game_id、Step、 pacmanPos、两只鬼位置、两只鬼 scared 状态、beans 和 energizers。 action_dir 和 available_dir 在 tile 修正后再生成。

输出字段

本阶段输出是后续主分析流程的基础字段集合；后续阶段会基于这些字段继续生成动作、utility、权重和特征字段。

字段含义类型
frame_id原始 frame 在排序后的稳定整数编号，替代旧字段 Unnamed: 0/frameIndex。int64
DayTrialtrial 标识，保留原始字符串形式，用作 trial id。str
game_id从 DayTrial 中去掉第二段 trial round 后得到的 game 标识。str
Step原始 trial 内 step 编号。int64
pacmanPosPacman 当前所在地图格点。tuple[int, int]
ghost1PosBlinky 当前所在地图格点。tuple[int, int]
ghost2PosClyde 当前所在地图格点。tuple[int, int]
ifscared1Blinky scared/dead 状态码；缺失或不可用状态用 -1。int8
ifscared2Clyde scared/dead 状态码；缺失或不可用状态用 -1。int8
beans当前地图中剩余普通豆子的格点列表；无豆子时为空列表。list[tuple[int, int]]
energizers当前地图中剩余 energizer 的格点列表；无 energizer 时为空列表。list[tuple[int, int]]

04. tile data 预处理

脚本 script/04_human_tile_data_preprocess.py

输入数据data/03_preprocessed_frame_data，迷宫邻接表 data/constant_data/adjacent_map_fmri.csv。
输出数据data/04_tile_data 和 data/04_corrected_tile_data。

目的

将逐 frame 数据转为 tile 级行为数据，并修正相邻位置缺失造成的路径断点。 该阶段生成动作字段，是后续 utility、拟合和特征计算的行为基础。

处理方式

1. 抽样生成 tile data

对每个 DayTrial 按 sample_rate 抽样，默认每 25 帧取一行。 每个 trial 的最后一帧会被强制保留，即使它不在固定采样间隔上。

2. 删除无效 tunnel 外坐标

Pacman 位于 (-1, 18) 或 (31, 18) 的记录会被删除。 这些位置是 tunnel 外的中间标记，不属于后续邻接表稳定覆盖的地图格点。

3. 修正路径连续性

corrected tile data 会检查相邻抽样点的 pacmanPos 是否连续。 若两个点之间缺少中间格，脚本会根据邻接路径插入缺失行，保证后续每一行到下一行都是一步动作。 左右 tunnel 的 (0, 18) 与 (30, 18) 按环形相邻处理。

4. 修正 ghost 特殊位置

ghost 位于家门口特殊坐标 (14, 20)、(15, 20)、(16, 20) 时， 会映射到门外可寻路位置，避免后续距离和邻接计算无法定位。

5. 生成 action_dir

action_dir 根据当前行和下一行的 pacmanPos 计算， 表示“从当前点走到下一点”的动作。最后一行没有下一步，动作缺失。

6. 生成 available_dir

在 action_dir 生成后，脚本读取 adjacent_map_fmri.csv 判断该动作是否是当前位置的合法方向。 当前字段是单个 bool，表示本行真实动作是否合法，不是四方向 bool 数组。

本阶段新增字段

字段含义类型
action_dir从当前行 pacmanPos 走到下一行 pacmanPos 的动作方向；最后一行或无法确定下一步时缺失。str 或缺失；取值 left/right/up/down
available_dir当前行真实 action_dir 在邻接表中是否为合法可走方向；这是单个布尔值，不是四方向数组。bool

05. 计算 utility

脚本 script/05_calculate_utility.py

输入数据data/04_corrected_tile_data，常量目录 data/constant_data。
输出数据data/05_utility_data。

目的

集中完成策略 utility 的计算、修正和归一化，避免拟合阶段再重复处理 Q 值。

处理方式

1. 准备稳定行标识

输入 corrected tile data 后，脚本为每行生成 row_id。 该字段标注当前表内行号，用于动态拟合、权重修正和后续特征回指。 本阶段之后的数据不会删除或重排行，因此 row_id 可以作为稳定引用。

2. 读取地图常量

从 data/constant_data 读取邻接表、可走路径、最短路径距离和全局方向区域。 邻接表用于判断墙方向和 tunnel 特例；距离表和路径快表用于路径搜索；全局方向区域用于 global_Q 的区域 bean 统计。

3. 计算 global_Q

global_Q 不展开路径树，而是使用当前位置预编译的四方向区域 mask。 对每个可走方向，计算该方向区域内当前剩余 bean 的数量；不可走方向暂时保持 0， 后续统一修正为 -np.inf。

global_Q[d] = count(bean_mask ∩ global_region_mask[pacmanPos, d])

4. 计算 6 个路径型策略 Q

local_Q、evade_blinky_Q、evade_clyde_Q、 approach_Q、energizer_Q、no_energizer_Q 使用同一个共享路径搜索引擎计算。引擎从当前 Pacman 位置出发，按 left/right/up/down 四个首步方向展开可走路径，并根据不同策略的搜索深度和奖励/风险规则累计叶节点 utility。

每个策略、每个首步方向的 Q 值不是取最大路径，而是取该首步方向下所有叶节点 utility 的平均值。 如果某方向没有叶节点，则该方向 Q 保持 0。

Q_strategy[d] = sum(leaf_utility_strategy[d, leaf]) / leaf_count_strategy[d]

默认搜索深度为：global 15、local 10、evade 10、approach 10、energizer 10、no_energizer 8。 默认随机扰动和行动惯性系数均为 0，因此当前流程输出是确定性的。

5. 策略 utility 的语义

local_Q 综合局部 bean/energizer 收益和两只鬼的风险； evade_blinky_Q、evade_clyde_Q 分别表示远离单只鬼的风险规避价值； approach_Q 表示在 ghost 可接近时靠近或吃 ghost 的价值； energizer_Q 强调获取 energizer 后带来的收益和风险变化； no_energizer_Q 表示在不吃 energizer 情况下追求普通 bean 的价值。

6. 修正不可走方向

根据当前位置的邻接表，将墙方向的 Q 写为 -np.inf。 这个修正只影响 Q 数组中的不可走方向，不改变任何行和非 Q 字段。

7. 处理 evade/no_energizer 的尺度

对 evade 和 no_energizer 类 Q，会先根据有限值最小偏移把可走方向平移到非负尺度， 再进入归一化；这样保留旧流程中这些策略的相对强度语义。

Q_available = Q_available - min(all finite values in this Q column)

8. 生成 Q_norm

对每个 raw Q 生成对应 *_Q_norm。归一化只在有限可走方向上计算， -np.inf 的墙方向不参与最大值计算。若有限方向全为 0，则保持 0。

Q_norm[d] = Q[d] / max(Q[finite directions])，墙方向继续为 -inf

本阶段新增字段

字段含义类型
row_id当前 utility 表内稳定行号；后续动态拟合和权重修正用它回指原行。int64
global_Q全局区域 bean 数量产生的四方向 utility。np.ndarray(shape=(4,), dtype=float64/float32)
local_Q局部 bean/energizer 收益与两只鬼风险共同产生的四方向 utility。np.ndarray(shape=(4,), dtype=float64)
evade_blinky_Q规避 Blinky 的四方向 utility。np.ndarray(shape=(4,), dtype=float64)
evade_clyde_Q规避 Clyde 的四方向 utility。np.ndarray(shape=(4,), dtype=float64)

no_energizer_Q不吃 energizer 情境下的四方向 utility。np.ndarray(shape=(4,), dtype=float64)

06. 动态策略拟合

脚本 script/06_dynamic_strategy_fitting.py

输入数据data/05_utility_data，邻接表 data/constant_data/adjacent_map_fmri.csv。
输出数据data/06_weight_data。

目的

根据每个 trial 中行为动作和七类策略 Q 值，拟合动态策略权重，并给出逐行预测方向和正确性。

处理方式

1. 校验拟合输入

输入必须已经包含七类 *_Q_norm、action_dir、 available_dir、DayTrial 和 row_id。 拟合阶段不再重新计算 Q，也不再重新推断动作方向。

2. 按 trial 切分行为序列

数据按 DayTrial 分组，每个 trial 独立切段。初始切点来自 action_dir 的变化位置：当相邻非缺失动作发生变化时形成候选切点， trial 末尾总是作为最后一个切点。

3. 加入事件切点并保护上下文

脚本会把吃 energizer 和吃 ghost 的位置加入切点。吃 energizer 根据 energizers 列长度下降识别；吃 ghost 根据 ifscared1/2 == 3 且相对上一行状态下降识别。连续缺失动作达到 stay_length=6 的片段会作为 stay/all-NaN 上下文保护，不被普通方向切点切开；吃 energizer 后到最后一次吃 ghost 的上下文也会被保护。

4. 合并过短或不可拟合片段

初始段落会按事件类型继续合并。全 NaN 段长度小于 stay_length 会并入相邻段； 普通段必须长度大于 3 且包含有效合法动作才独立拟合；事件段如果没有有效合法动作也会合并。 合并后每个段落用全局 row_id 范围表示，并写入 trial_context。

5. 构造段落内拟合样本

对每个段落，只使用 action_dir 非缺失且 available_dir=True 的行拟合。 真实动作被转为四方向 one-hot；七个 *_Q_norm 被整理为形状 (样本数, 4, 7) 的数组。

6. 使用遗传算法搜索权重

每段数据用遗传算法搜索七维策略权重。默认种群大小为 100，迭代 500 次， 变异概率为 0.01，权重精度为 0.001，并带有 L1 权重惩罚。 并行时可用 --segment-workers 加速单文件内部段落拟合。

段落内给定权重 w 后，先对每行、每个方向计算七类 Q_norm 的加权和， 再选择分数最大的方向作为预测方向。若多个方向并列最大，正确率按并列方向数分摊。

score[i, d] = Σ_k Q_norm[i, d, k] * w[k]
pred[i] = argmax_d score[i, d]
objective(w) = -accuracy(w) / n + 0.1 * Σ_k |w[k]|，约束 0 ≤ w[k] ≤ 1

7. 归一化权重并判断 vague

拟合得到的 weight 是 GA 输出的原始权重。非 stay 且权重和非 0 的段落会再做段内 min-max 归一化，得到 normalized_weight。如果单一最大策略的预测正确率不高于 0.51，该段被标为 is_vague=True。

normalized_weight = (weight - min(weight)) / (max(weight) - min(weight))

8. 预测方向并判断正确性

拟合得到的权重与七类 Q_norm 加权后形成四方向分数。 分数最高方向写入 predict_dir，方向编码为 0/1/2/3，分别对应 left/right/up/down。与真实 action_dir 比较后写入 prediction_correct；无有效动作的行保持缺失。

9. 标注 trial 上下文

输出保留 trial_context、eat_energizer、eat_ghost。 连续停留达到 stay_length 的片段标为 is_stay； 单策略正确率低于阈值时标为 is_vague。

本阶段新增字段

字段含义类型
weight段落级 GA 拟合得到的七策略原始权重，顺序为 global、local、evade_blinky、evade_clyde、approach、energizer、no_energizer。list/np.ndarray[float]，长度 7

07. 修正人类策略权重

脚本 script/07_revise_human_weight.py

输入数据data/06_weight_data。
输出数据data/07_corrected_weight_data。

目的

根据 energizer、ghost、scared 时间窗口和 stay/vague 等规则，对拟合后的归一化权重和预测结果进行修正。

处理方式

1. 按 trial 读取拟合结果

2. 根据事件上下文修正权重

4. 生成策略编号

本阶段新增字段

10. 生成状态依赖图

目的

4. 写出每个被试的图结构

11. 生成 grammar

脚本 script/11_generate_grammar.py

输入数据data/09_strategy_sequence 和 data/10_state_dependency_graph_data。
输出数据data/11_grammar。

目的

基于策略序列和状态依赖图，学习每个被试的 grammar chunk、解析结果和 skip-gram 信息。

处理方式

1. 配对读取序列和状态图

对每个被试文件，脚本读取 09_strategy_sequence 中的策略序列， 以及 10_state_dependency_graph_data 中同名状态图。两者共同决定 chunk 候选的统计和约束。

2. 初始化基础 grammar

初始 token 来自策略符号，例如 G、L、1、2、 A、E、N、S、V。 参数中记录状态名、学习系数、最大迭代次数、收敛窗口和候选保留阈值。

3. 迭代学习 chunk 候选

学习过程根据策略序列中相邻 token 的组合频率、状态依赖约束和候选得分生成 chunk。 达到收敛条件或最大迭代次数后停止。--quiet 只影响日志，不影响结果。

4. 解析序列并保存结果

输出对象包含 grammar、parsed 和 skip_gram。 grammar 中 components 第二项非空的条目就是实际学到的 chunk， 例如 E-A=E+A 或 E-A-G-L=E-A+G-L。

5. 记录来源和参数

source 保存输入文件和参与者集合，parameters 保存学习参数。 这些元数据用于解释结果来源，不参与下游数值计算。

输出对象字段

字段含义类型
source输入 strategy sequence、状态依赖图和被试来源信息。dict
parametersgrammar 学习参数，包括状态名、学习系数、最大迭代次数、收敛窗口和候选阈值。dict
grammar学习得到的 grammar token/chunk 列表；其中 components 非空的条目表示组合 chunk。list
parsed策略序列被 grammar 解析后的结果。dict
skip_gram跳跃组合相关的 grammar 信息。dict
