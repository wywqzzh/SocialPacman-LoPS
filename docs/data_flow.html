Social Pacman 正式数据流程

仓库只保留一条 01–07 正式流程。各阶段使用嵌套的 data/<stage>/<task>/<session>.pkl，单人文件不补空 P2 字段。

关键处理

02：修正 Pacman/Ghost 坐标、统一 frame 字段，并整 trial 删除出现 Ghost3/Ghost4 的非 two-ghost 数据。

03：选择正式分析字段并统一类型，不再执行 trial 过滤。

04：先保留 P1/P2 坐标与 mode、两只 Ghost 坐标的联合状态变化候选帧，再在 13 frame 异步切换窗口内删除不破坏全对象连续性的中间候选，并重算动作方向。

05：计算七种正式策略的四方向 raw Q，并额外保存三类目标候选。正式 Global 是四方向区域内距离大于 10 的普通豆计数；Cluster Global 将普通豆和 Energizer 按地图最短路聚类。Energizer 候选按单颗能量豆计算，Approach 候选按单只非 dead Ghost 计算。正式 Q 和候选 utility 不是同一算法。

06：为每位玩家独立检测事件并划分 context，在每段选择解释动作最好的目标， 对合法方向 Q 做 Min-Max，使用 softmax likelihood 拟合文件级 beta 并计算策略 posterior。

07：不覆盖模型 posterior；结合事件结果和单策略方向证据，追加 revised score、 revised strategy 和诊断字段。

视频

PYTHONPATH=src python script/pacman_video/run_tile_video_renderer.py \
  --task comp --session <session>.pkl --trial <DayTrial> --save-frames

默认读取 data/07_revised_strategy_data，MP4 输出到 data/pacman_video/tile_video/<task>/，逐帧 PNG 输出到 data/pacman_video/tile_frame_images/<task>/<session>/<trial>/。 帧号统一使用 0-based，context 显示为闭区间。

地图准备

map_constants.pkl 不随仓库分发。首次运行 04、05 或视频前执行：

PYTHONPATH=src python script/constant_map/generate_map_constants.py --workers 8

生成器需要 networkx；视频入口还需要 imageio 和 imageio-ffmpeg。这些依赖当前未写入 pyproject.toml。
