04 的实际语义

04_tile_data 在每个 trial 内比较 P1/P2 的位置与 mode，以及 ghost1、 ghost2 的位置。第一帧保留；之后只有这些联合采样字段发生变化的帧才保留。 ifscared、豆子和 energizer 变化本身不触发候选帧。

04_corrected_tile_data 以默认 13 帧作为异步切换窗口，在不破坏所有对象 空间连续性和 Pacman 独有短暂 mode 状态的前提下删除冗余中间候选。它不采用 “每 25 帧取一行”，也不 插入推测的中间路径点。动作字段为 p1_action_dir/p2_action_dir，合法性字段为 p1_available_dir/p2_available_dir；单人文件不补 P2 列。

04 读取的是 constant_data/map_constants.pkl 中的 adjacent_map， 不是旧说明中的独立 adjacent_map_fmri.csv。更详细的压缩理由见 docs/04_tile_sampling_issue_solution.md。

05 / 05b：Utility 分支

05 Cluster-global hierarchical utility（当前默认）

script/05_calculate_utility.py 默认读取 04_corrected_tile_data， 输出到 05_cluster_global_utility_data。每位玩家保存七种策略的四方向 raw Q 和 Q_norm；Global 还保存资源团候选矩阵及 meta，供 06b/06c 在 context 内选择 best Global。

参数默认值含义
global_cluster_radius60可进入 Cluster Global 的最远资源团距离。
global_cluster_distance_threshold2资源点聚类的地图距离阈值；允许豆子之间存在一个空格。
local_depth10Local 路径树深度。
evade_depth6正常 ghost 的即时威胁搜索深度。
approach_depth20追逐非危险 ghost 的搜索深度。
energizer_depth / no_energizer_depth10 / 8两种 energizer 相关路径深度。
randomness_coeff / laziness_coeff0 / 0默认关闭扰动与惯性，保证可复现。

2026-07-11 已修复 ghost 状态被有限 float 误判为缺失，以及 Q_norm 生成过程原地改写 raw Q 的问题。详细数值影响和验证见 docs/05_utility_numeric_bug_audit.md。

05b Range utility（独立实验）

script/05b_calculate_range_utility.py 默认输出 05_range_utility_data， 使用地图距离、策略半径、衰减系数和奖励/惩罚计算方向 utility。它不是 05 的别名， 继续进入 06 类脚本时必须显式指定输入与输出目录。

06 / 06b / 06c：三种拟合接口

入口默认目录模型与 context
06_dynamic_strategy_fitting.py 05_utility_data → 06_weight_data 历史动态切段与 GA 七策略权重；默认 seed 20260610、GA 种群 100、迭代 500。
06b_fit_dynamic_strategy_event_context.py 05_cluster_global_utility_data → 06_cluster_global_event_context_weight_data 事件硬边界、掉头/队友事件软边界、context 内 best Global，仍使用 GA 权重。
06c_fit_context_strategy_posterior.py 05_cluster_global_utility_data → 06c_context_strategy_posterior_data 沿用 06b context/best Global，改为七个潜在策略的 context posterior 和文件级 beta。

06b/06c 的 context 边界

  • trial 起止、当前玩家生死变化、吃 energizer、吃 ghost 是强边界。
  • 连续吃普通豆的首尾是可抑制弱硬边界；强事件后 3 行内的吃豆开始、强事件前 3 行内的吃豆结束会被取消。
  • 长度至少 4 的 stay 默认形成硬边界；当前玩家吃 ghost 前后 5 行内的 stay 不再额外切段。
  • 掉头和队友吃 energizer/ghost 是软边界；软边界产生的短段只在同一硬区间内合并。
  • 普通转向、玩家到 bean 的地图距离、ghost 计时恢复不再生成 context 边界。

06c-v3 posterior

每个玩家、tile、策略都从 raw Q 重新执行合法方向 Min-Max。策略在一个 context 内 能区分合法方向的有效动作比例称为信息覆盖率；默认覆盖率至少 0.50 才能参与 beta 和 posterior。合格策略内部使用均匀先验。posterior 阈值默认为 0.70。

文件内比较 P1/P2 共享一个 beta 与各自一个 beta 的结构，并用有效 player-context 数量计算 BIC。按完整 DayTrial 做最多 5 折 grouped CV；CV 只用于诊断， 最终参数仍由全文件重拟合和 BIC 决定。beta 默认搜索范围为 [0.05, 20]，网格大小 81。

Null 模型是在合法方向上均匀随机动作的诊断基线，不参与 beta、BIC、posterior 或 候选策略。context 有动作但所有策略 coverage 均不足时，posterior 为七个 0、 candidate 为 none、策略标签为 vague。

07 / 07c：规则修正

07 历史 GA 修正

07_revise_human_weight.py 默认读取 06_weight_data。它在 joint-state 表中分别映射 P1/P2 的 normalized weight、私有事件和 Q_norm，执行 vague、energizer、 scared-time 等人工规则，并新增玩家前缀的 revised 字段与策略编号。06b 输出可通过显式 --input-dir 传给该入口。

07c posterior 修正

07c_revise_context_strategy_posterior.py 默认读取 06c，输出版本为 07c-v6。它保留原始 strategy_posterior、strategy 和模型 attrs， 将 posterior 仅作为人工规则的初始策略分数，结果写入 revised_strategy_score/revised_strategy/revised_strategy_name 等新字段。

revised score 可能是 one-hot 或 multi-hot，不是概率，不要求和为 1。当前“未由本玩家 吃到 ghost 就否定 Approach”的二次修正规则已停用；attrs 会记录 legacy_rule_order_reused=False 与 uneaten_ghost_approach_revision_enabled=False。

视频流程

逐 frame 渲染

02_frame_data + grammar_data→ run_render_table.py→ run_frame_renderer.py→ run_video_renderer.py

该链路用于逐帧展示，最终合成 MP4 需要系统可调用 ffmpeg。

Tile 结果检查

run_tile_video_renderer.py 直接读取 07 类 tile 表和 constant_data/map_constants.pkl，不依赖 render table 或 grammar。其默认 tile root 仍为历史 07_corrected_weight_data；查看 07c 时必须显式指定 --tile-root data/07c_context_strategy_posterior_corrected_data。

结果有效性与版本边界

  • 目录存在或 pickle 数量正确，只说明文件被写出，不说明它由当前代码生成。
  • 05 数值修复后必须按同一输入重跑 05，再依次重跑 06c 和 07c。
  • 06c 输出 attrs 的版本应为 06c-v3，07c 应为 07c-v6。
  • 06c 应记录信息覆盖率阈值、epsilon、beta 模型、BIC、CV 和 Null 仅诊断标志。
  • 验证输出写入 data/validation，不写入 .planning。
当前方法细节以 docs/context_strategy_posterior_inference.md 为准；05 修复证据 以 docs/05_utility_numeric_bug_audit.md 为准。
